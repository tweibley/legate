Okay, that's an interesting idea! Using Gemini itself to generate example inputs based on the agent's configuration could be quite helpful.
Here's a breakdown of the necessary changes and a plan to implement this:
Understanding the Goal:
We want to add a button/link near the "Execute Task (JSON Input)" area. When clicked, it should:
Send a request to the backend with the current agent's name.
The backend will determine the agent's configured tools and their parameters.
The backend will use this tool information to prompt a Gemini model to generate one or more example JSON inputs demonstrating how to use those tools (specifically focusing on the parameters).
The generated example JSON will be sent back to the frontend.
The frontend JavaScript will update the "Execute Task" CodeMirror editor with the generated example.
Research / Code Analysis:
Tool Metadata: We already have the logic in GET /agents/:name (app.rb) to fetch configured tool names from Redis and gather detailed metadata (description, parameters, source) for both native and MCP tools into the all_available_tools_metadata_map. We can reuse this heavily.
Gemini API Client: The application already uses Gemini for planning (via Agent and Planner classes). We need to find how the client is initialized and make calls (likely using a library like google-cloud-ai_platform or google-generative_ai configured via environment variables). We'll need to leverage this existing mechanism. Let's assume an appropriate client (gemini_client) can be accessed or created.
Execute Task Input: Currently, the POST /agents/:name/execute endpoint expects JSON like {"task": "user description"} and passes the description to agent.run_task. Generating example parameter JSON won't directly fit this flow. The generated JSON will serve as an example for the user to adapt or understand the structure, rather than being directly executable by the current "Execute Task" button. We'll simply populate the editor with the example.
Plan:
Phase 1: Backend Changes (in lib/adk/web/app.rb)
New Route:
Define a new route: GET /agents/:name/generate_example_task.
Agent Config & Tool Metadata:
Inside the route handler, get name from params.
Fetch agent configuration from Redis (especially the tools field).

Fetch all available native tool metadata (GlobalToolManager.list_all_tools).
Fetch all available MCP tool metadata (using fetch_mcp_tools and SchemaConverter).
Create the combined map (all_available_tools_metadata_map).
Filter this map using the agent's configured_tool_names to get a list/hash (configured_tools_metadata) containing only the metadata for tools this agent actually has. Handle the case where there are no configured tools.

Create a detailed prompt string. It should instruct Gemini to generate example JSON parameter structures for one or more tools from the provided list.

Instructions for Gemini: Specify the desired output format clearly (e.g., "Generate a JSON object representing example parameters for the 'calculator' tool.
